#Репозиторий - место, где хранится код и другие файлы проекта

#Commit - запись изменений в репозитории

#Branch(ветка) - парарельная версия репозитория которая позволяет изменить код

#Слияние (merge) - процесс объяденения изменений из одной ветки с другой веткой

#Конфликт слияния(merge conflict) - ситуация когда две ветки содержат разные версии одного и того же файла.
# И Git не может автомотически решить какую версию использовать

#Вилка(fork) - копия репозитория, которая создаётся, когда пользователь хочет внести в проект, но не имеет
#доступа к оригинальному репозиторию

#Клонирование(clone) - процесс создание локальной копии удалённого репозитория на пк пользователя

#Index - структура данных которая хранит информацию о том какие файлы будут включены в следующий commit

#Откат(revert) - процесс отмены изменений, сделанных в определённом commit'е

#Удалённый(далекой) репозиторий(remote repository) - репозиторий который расположен на удалённом сервере и доступен для других пользователей

#Pull - запрос(Pull request) - запрос на внесение изменений в оригинальный репозиторий, который создаётся после внесений изменений в вилку

#Орг. репозиторий (upstream repository) - репозиторий, который был форкнут(украден) и используется в качестве осовного для внесения изменений

#История commit'ов(commit history) - список всех commit'ов, сделанных в репозитории в хронологическом порядке

#Teg - ссылка на конкретный commit

#Stage - это временное хранилище изменений, который ещё не готовы к commit'у

#Git ignore - файл, который указывает Git какие файлы и папки должны быть в ignore

#Хук(hook) - скрипт, который запускается автоматически в определённых событиях Git, например, перед или после commit

#Sub module - отдельных репозиторий, который используется как подмодуль в основном репозитории
